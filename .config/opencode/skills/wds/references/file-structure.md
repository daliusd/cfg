# WDS Docs File Structure

This reference provides technical details about the structure and sizes of WDS documentation files.

## File Formats

### Components Overview (components.md)

```markdown
### ComponentName
- description: <p>What the component does...</p>
- do: When to use it
- donts: When NOT to use it (or "No donts")
```

**Pattern:** Component name as heading, followed by bullet points for metadata.

### Props Files ({ComponentName}Props.md)

```markdown
### propName
- type: TypeDefinition
- description: What the prop does

### anotherProp
- type: string | number
- description: Another prop description
```

**Pattern:** Every prop is a heading (###), followed by type and description bullets. Typically 3 lines per prop.

### Example Files ({ComponentName}Examples.md)

```markdown
## Feature Examples

### ExampleName                    ← Line N (grep "^### " to find)
- description: What it demonstrates
- example:
```jsx
<Component prop="value">          ← Start reading from line N
  Content
</Component>
```                                ← Example ends ~20-50 lines later

### NextExample                    
- description: Next feature demo
- example:
```jsx
...
```
```

**Pattern:** Each example is a heading (###) with description and code block. Examples are spaced 20-50 lines apart.

## File Sizes Reference

| File | Lines | Read Strategy |
|------|-------|---------------|
| `components.md` | ~970 | **Always grep**, never read fully |
| `icons.md` | ~400 | **Always grep** for specific icon names |
| `ButtonProps.md` | ~50 | Read fully OK |
| `InputProps.md` | ~80 | Read fully OK |
| `CardProps.md` | ~60 | Read fully OK |
| `BoxProps.md` | **8000+** | ⚠️ **Grep only!** Never read this file |
| `LayoutProps.md` | ~200 | Grep specific props |
| `TableProps.md` | ~150 | Grep specific props |
| `ButtonExamples.md` | ~300 | Grep → offset read |
| `CardExamples.md` | ~250 | Grep → offset read |
| `PageExamples.md` | ~940 | Grep → offset read |
| `TableExamples.md` | ~600 | Grep → offset read |

### Size Categories

**Small (<100 lines)** - Safe to read fully:
- Most simple component Props files (Button, Input, Checkbox, etc.)

**Medium (100-300 lines)** - Prefer grep, read if needed:
- Complex component Props files (Table, Modal, Dropdown)
- Most Example files

**Large (300-1000 lines)** - Always grep, use offset reading:
- components.md
- icons.md
- Large Example files (Page, Table)

**Huge (1000+ lines)** - Never read, grep only:
- BoxProps.md (8000+ lines!)
- Any autogenerated type files

## Common Grep Patterns

### Finding Components by Feature

```bash
# Forms
Grep: "form\|input\|validation" in node_modules/@wix/design-system/dist/docs/components.md

# Layout
Grep: "layout\|page\|card\|box" in node_modules/@wix/design-system/dist/docs/components.md

# Data display
Grep: "table\|list\|badge" in node_modules/@wix/design-system/dist/docs/components.md

# Feedback
Grep: "notification\|toast\|loader" in node_modules/@wix/design-system/dist/docs/components.md

# Navigation
Grep: "breadcrumb\|pagination\|menu\|sidebar" in node_modules/@wix/design-system/dist/docs/components.md
```

### Finding Specific Props

```bash
# Boolean props
Grep: "### disabled\|### required\|### readOnly" in components/InputProps.md -A 2

# Size variants
Grep: "### size" in components/ButtonProps.md -A 2

# Event handlers
Grep: "### onChange\|### onClick\|### onFocus" in components/InputProps.md -A 2
```

### Listing All Examples

```bash
# Get all example names with line numbers
Grep: "^### " in node_modules/@wix/design-system/dist/docs/components/ButtonExamples.md -n

# Output format:
# 5:### Size
# 17:### Skin
# 71:### Affix
```

### Reading Specific Examples

```bash
# Method 1: Read with offset (preferred)
Read: node_modules/@wix/design-system/dist/docs/components/ButtonExamples.md offset=71 limit=40

# Method 2: Grep with context
Grep: "### Affix" in node_modules/@wix/design-system/dist/docs/components/ButtonExamples.md -A 40
```

## High-Traffic Components

Learn these patterns first (most commonly used):

### Essential Components (~30% of usage)
- Button, Input, FormField, Dropdown
- Card, Box, Layout, Page
- Text, Heading

### Frequent Components (~40% of usage)
- Table, Modal, Tabs
- Checkbox, RadioGroup, ToggleSwitch
- Badge, Avatar, Loader

### Specialized Components (~30% of usage)
- DatePicker, AutoComplete, RichTextInputArea
- SidePanel, Accordion, Breadcrumbs
- ColorPicker, ImageViewer, Calendar

## TypeScript Type Files

Located at: `node_modules/@wix/design-system/dist/types/`

```
types/
├── index.d.ts                    # All type exports
├── Button/
│   └── index.d.ts               # Button types
├── Input/
│   └── index.d.ts               # Input types
└── ...
```

**Usage:**
```bash
# Get all exported types
Read: node_modules/@wix/design-system/dist/types/index.d.ts

# Get component-specific types
Read: node_modules/@wix/design-system/dist/types/Button/index.d.ts
```

Most type information is already in the Props.md files, so only read these when you need:
- Generic type parameters
- Complex union types
- Type utilities and helpers
- Internal type definitions

## Documentation Update Frequency

WDS documentation is generated from the installed package version. If docs seem outdated:

```bash
# Check installed version
Read: node_modules/@wix/design-system/package.json

# Update to latest
npm update @wix/design-system
```

The dist/docs/ folder is regenerated on package install, so it always matches the installed version.
